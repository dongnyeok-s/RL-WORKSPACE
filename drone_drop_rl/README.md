# 드론 타겟 투하 시스템 — 강화학습 보상 설계 및 알고리즘 제안서

## 1. 개요

드론이 타겟을 인식하고 이동하여 물체를 정확히 투하하는 시스템에 강화학습(RL)을 도입합니다.
이 문서는 **어떤 알고리즘을 왜 선택했는지**, 그리고 **보상 함수를 어떻게 설계했는지**를 정리합니다.

---

## 2. 알고리즘 선택: SAC (Soft Actor-Critic)

테스트한 4개 알고리즘 중 **SAC**를 주력 알고리즘으로 선택했습니다.

### 왜 SAC인가?

| 기준 | SAC | PPO | 왜 중요한가 |
|------|-----|-----|-------------|
| **데이터 재사용** | 한 번 수집한 데이터를 수천 번 재학습 | 한 번 쓰고 버림 | 드론 비행은 비용이 크므로 데이터 효율이 핵심 |
| **연속 제어** | 속도·방향을 부드럽게 제어 | 가능하지만 불안정 경향 | 드론은 연속적인 속도 제어가 필요 |
| **탐색 전략** | 자동으로 탐색 강도 조절 | 수동 설정 필요 | 하이퍼파라미터 튜닝 부담 감소 |

**핵심**: SAC는 Off-Policy 알고리즘이라 과거 경험을 버퍼에 저장해두고 반복 학습합니다.
PPO는 On-Policy라 매번 새 데이터를 수집해야 합니다.
실제 드론은 배터리·안전·시간 제약이 크므로, 적은 데이터로 많이 학습할 수 있는 SAC가 유리합니다.

## 3. 보상 함수 설계

### 핵심 질문: "투하 결과만 보상하면 안 되나?"

안 됩니다. 이유는 다음과 같습니다:

```
[문제] 투하 결과만 보상 (Sparse Reward)
  ┌─────────────────────────────────────────────────┐
  │  스텝 1    스텝 2    ...    스텝 N    투하!      │
  │   0점       0점      ...    0점      15점       │
  └─────────────────────────────────────────────────┘
  → 에이전트 입장: "어떤 행동이 좋았는지 모르겠다" (학습 느림, 불안정)

[해결] 4계층 보상 (Shaped Reward)
  ┌─────────────────────────────────────────────────┐
  │  스텝 1    스텝 2    ...    스텝 N    투하!      │
  │  +0.2      +0.1     ...    +0.3     15점       │
  │  접근↑     정렬↑    ...    타이밍↑  정밀도↑     │
  └─────────────────────────────────────────────────┘
  → 에이전트 입장: "타겟에 가까워지면 좋고, 정렬하면 좋고, 정확히 투하하면 최고"
```

### 4계층 보상 구조

```
보상 = Layer 1 (안전) + Layer 2 (효율) + Layer 3 (접근) + Layer 4 (정밀도)

        ┌─────────────────────────────────────────────────┐
        │                                                 │
        │   Layer 4: 투하 정밀도  ← 에피소드 1회, 최대 25점  │
        │   ┌─────────────────────────────────────────┐   │
        │   │  거리 비례 보상 (최대 15점)              │   │
        │   │  + 정밀도 보너스 (1m: +10, 3m: +5, 5m: +2) │ │
        │   └─────────────────────────────────────────┘   │
        │                                                 │
        │   Layer 3: 접근 유도    ← 매 스텝, 투하 전       │
        │   ┌─────────────────────────────────────────┐   │
        │   │  타겟에 가까워지면 +, 멀어지면 -          │   │
        │   │  + 비행경로 정렬 보너스                   │   │
        │   │  + 최적 투하 지점 근접 시 타이밍 보상      │   │
        │   └─────────────────────────────────────────┘   │
        │                                                 │
        │   Layer 2: 효율성       ← 매 스텝               │
        │   ┌─────────────────────────────────────────┐   │
        │   │  시간 페널티: -0.01/스텝 (빠른 완수 유도) │   │
        │   │  에너지 페널티: 급격한 조작 방지           │   │
        │   └─────────────────────────────────────────┘   │
        │                                                 │
        │   Layer 1: 안전         ← 항상 활성             │
        │   ┌─────────────────────────────────────────┐   │
        │   │  최소 고도 위반: -10 + 즉시 종료          │   │
        │   │  최대 속도 위반: -8 + 즉시 종료           │   │
        │   └─────────────────────────────────────────┘   │
        │                                                 │
        └─────────────────────────────────────────────────┘
```

### 보상 요약표

| 구성요소 | 크기 | 발생 시점 | 목적 |
|----------|------|----------|------|
| 안전 위반 | -10 | 위반 시 | 위험한 행동 방지 |
| 시간 페널티 | -0.01/스텝 | 매 스텝 | 빠른 임무 완수 |
| 에너지 페널티 | ~-0.005/스텝 | 매 스텝 | 부드러운 비행 |
| 접근 보상 | ±0.3 | 매 스텝 | 타겟으로 유도 |
| 정렬 보너스 | 0~0.02 | 매 스텝 | 비행경로 품질 |
| 타이밍 보상 | +0.1 | 최적 지점 근처 | 투하 타이밍 학습 |
| **거리 보상** | **0~15** | **투하 시 1회** | **핵심 목표** |
| **정밀도 보너스** | **+2~+10** | **투하 시 1회** | **정확도 인센티브** |

### 보상 해킹 방지

에이전트가 보상 시스템의 허점을 이용하는 것을 방지합니다:

1. **접근 보상**: Potential-based Shaping 사용 — 타겟 근처를 맴돌아도 추가 보상 없음 (수학적 보장)
2. **에너지 페널티**: 불필요한 진동·급격한 방향 전환 방지
3. **1회 투하**: 한 에피소드에 한 번만 투하 가능 — 반복 시도 불가

## 4. Sparse vs Shaped 보상 비교 실험

같은 SAC 알고리즘, 같은 환경에서 보상만 다르게 학습하여 비교합니다:

| 실험 | 보상 구성 | 기대 결과 |
|------|----------|----------|
| **SAC + Sparse** | 투하 시 거리 보상만 | 학습 느림, 성공률 낮음 |
| **SAC + Shaped** | 4계층 전체 | 학습 빠름, 성공률 높음, 안정적 |

비교 지표:
- **학습 속도**: 목표 성능에 도달하는 데 걸리는 스텝 수
- **최종 성공률**: 5m 이내 착지 비율
- **학습 안정성**: 보상 분산 (작을수록 안정적)

실행 방법:
```bash
# Shaped 보상으로 학습
python -m drone_drop_rl.train.train_sim --reward shaped

# Sparse 보상으로 학습 (대조 실험)
python -m drone_drop_rl.train.train_sim --reward sparse

# 비교 차트 생성
python -m drone_drop_rl.compare
```

## 5. 커리큘럼 학습

난이도를 단계적으로 올려 학습 효율을 높입니다:

```
Stage 1 (Easy)          Stage 2 (Medium)        Stage 3 (Hard)
  타겟: 10m 반경          타겟: 5m 반경           타겟: 3m 반경
  바람: 없음              바람: 0-2 m/s           바람: 0-5 m/s
  거리: 20-40m            거리: 40-100m           거리: 50-150m

       ──80% 성공──→           ──80% 성공──→
```

각 단계에서 최근 100 에피소드 성공률이 80%를 넘으면 자동으로 다음 단계로 승급합니다.

## 6. 향후 확장: ROS2 + YOLO 연동

현재 시뮬레이션에서 학습한 모델은 나중에 ROS2 시스템에 통합할 수 있습니다.
YOLO로 타겟 좌표를 인식하여 관측 벡터에 피딩하고, 학습된 정책의 출력을
드론 제어 명령으로 변환하면 됩니다. Gymnasium의 obs/action 인터페이스가
그대로 ROS2 노드의 입출력으로 매핑됩니다.
